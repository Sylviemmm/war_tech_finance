# 战争技术金融可视化分析平台 - 项目结构

## 项目概述
基于FastAPI + SQLAlchemy + SQLite的战争数据可视化分析平台，支持COW和MID数据集。

## 数据库模型

### 核心实体
1. **War (战争)** - 战争/争端数据
2. **Country (国家)** - 国家信息
3. **Technology (技术)** - 军事技术
4. **Finance (金融)** - 战争融资数据
5. **Event (事件)** - 相关历史事件
6. **Relation (关系)** - 实体间关系

## API端点

### 战争API (`/api/v1/wars`)
- `GET /` - 获取战争列表（支持筛选、分页）
- `GET /{war_id}` - 获取战争详情
- `POST /` - 创建战争
- `PUT /{war_id}` - 更新战争
- `DELETE /{war_id}` - 删除战争
- `GET /detailed` - 获取详细战争数据（用于可视化）
- `GET /stats/summary` - 战争统计信息

### 国家API (`/api/v1/countries`)
- `GET /` - 获取国家列表
- `GET /{country_id}` - 获取国家详情
- `POST /` - 创建国家
- `PUT /{country_id}` - 更新国家
- `DELETE /{country_id}` - 删除国家

## 使用方法

### 1. 启动后端服务
```bash
cd "vibe coding"
python main.py
```
服务将在 http://localhost:8000 启动

### 2. 处理数据
```bash
# 处理MID数据
python process_mid_data.py

# 处理COW数据
python process_cow_data.py
```

### 3. 查看可视化
打开 `mid_war_tencent_map.html` 或 `mid_war_visualization_offline.html`

## 配置说明

### 腾讯地图API密钥
在HTML文件中替换 `YOUR_KEY`：
```html
<script src="https://map.qq.com/api/gljs?v=1.exp&key=YOUR_KEY&libraries=visualization"></script>
```

### 数据库配置
在 `config.py` 中修改：
```python
database_url: str = "sqlite:///./war_tech_finance.db"
```

## 数据说明

### MID数据集 (Militarized Interstate Disputes)
- 时间范围：1816-2014年
- 总争端数：2,436个
- 包含：争端信息、参与者、地理坐标

### COW数据集 (Correlates of War)
- 时间范围：1800-2007年
- 包含：国家间战争、国内战争、非国家战争

## 技术栈

- **后端**: FastAPI, SQLAlchemy, SQLite, Pydantic
- **前端**: HTML5, Tailwind CSS, JavaScript
- **地图**: 腾讯地图JSAPI GL, 个性化地图样式
- **数据处理**: pandas, numpy
