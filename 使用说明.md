# 战争技术金融可视化分析平台 - 使用说明

## 项目概述

本项目基于产品文档和数据筛选标准文档，构建了一个支持战争-技术-金融同构逻辑研究的可视化分析平台。项目采用Python + FastAPI + SQLite3技术栈，提供了完整的数据库设计和API接口。

## 技术栈

- **后端框架**: FastAPI (Python高性能框架)
- **数据库**: SQLite3 (单文件存储，便于学术研究)
- **ORM**: SQLAlchemy (Python数据库ORM)
- **数据验证**: Pydantic (FastAPI内置数据验证)
- **API文档**: Swagger/OpenAPI 自动生成
- **数据导入**: 支持CSV和JSON格式批量导入

## 数据库设计

### 核心表结构

#### 1. 国家表 (countries)
- **id**: 主键
- **name**: 国家名称
- **iso_code**: ISO国家代码（唯一）
- **iso3_code**: ISO3国家代码
- **continent**: 大洲
- **region**: 地区
- **political_system**: 政治体制
- **gdp**: GDP
- **military_spending**: 军费开支
- **population**: 人口数量
- **latitude**: 纬度
- **longitude**: 经度
- **created_at**: 创建时间
- **updated_at**: 更新时间

#### 2. 战争表 (wars)
- **id**: 主键
- **name**: 战争名称
- **start_date**: 开始日期
- **end_date**: 结束日期
- **duration_days**: 持续天数（自动计算）
- **belligerents**: 参战方（JSON格式）
- **casualties**: 伤亡人数
- **geographic_scope**: 地理范围（JSON格式）
- **war_type**: 战争类型
- **casus_belli**: 战争原因
- **impact_level**: 影响程度（1-10）
- **created_at**: 创建时间
- **updated_at**: 更新时间

#### 3. 技术表 (technologies)
- **id**: 主键
- **name**: 技术名称
- **invention_date**: 发明日期
- **inventor**: 发明者
- **application_domain**: 应用领域
- **military_civilian**: 军事/民用属性
- **patent_number**: 专利号
- **description**: 描述
- **significance_level**: 重要性级别（1-10）
- **created_at**: 创建时间
- **updated_at**: 更新时间

#### 4. 金融数据表 (finance_data)
- **id**: 主键
- **indicator_name**: 指标名称
- **value**: 数值
- **date**: 日期
- **country_id**: 国家ID（外键）
- **data_type**: 数据类型
- **source**: 数据来源
- **impact_level**: 影响程度（1-10）
- **created_at**: 创建时间

#### 5. 事件表 (events)
- **id**: 主键
- **name**: 事件名称
- **event_date**: 事件日期
- **event_type**: 事件类型
- **description**: 描述
- **related_entities**: 关联实体（JSON格式）
- **impact_score**: 影响分数（1-10）
- **created_at**: 创建时间
- **updated_at**: 更新时间

#### 6. 关系表 (relations)
- **id**: 主键
- **source_type**: 源实体类型
- **source_id**: 源实体ID
- **target_type**: 目标实体类型
- **target_id**: 目标实体ID
- **relation_type**: 关系类型
- **strength**: 关系强度（1-10）
- **start_date**: 开始日期
- **end_date**: 结束日期
- **created_at**: 创建时间

## API接口

### 基础信息
- **API地址**: http://localhost:8000
- **API文档**: http://localhost:8000/docs
- **ReDoc文档**: http://localhost:8000/redoc

### 核心接口

#### 1. 国家管理接口
```
GET    /api/v1/countries           # 获取国家列表（支持分页、过滤）
GET    /api/v1/countries/{id}     # 获取国家详情
POST   /api/v1/countries           # 创建国家
PUT    /api/v1/countries/{id}     # 更新国家
DELETE /api/v1/countries/{id}     # 删除国家
```

#### 2. 战争数据接口
```
GET    /api/v1/wars                # 获取战争列表（支持时间范围、类型过滤）
GET    /api/v1/wars/{id}          # 获取战争详情
POST   /api/v1/wars                # 创建战争记录
PUT    /api/v1/wars/{id}          # 更新战争记录
GET    /api/v1/wars/stats          # 战争统计数据
```

#### 3. 数据导入接口
```
POST   /api/v1/import/countries   # 批量导入国家数据
POST   /api/v1/import/wars       # 批量导入战争数据
POST   /api/v1/import/technologies # 批量导入技术数据
POST   /api/v1/import/finance     # 批量导入金融数据
POST   /api/v1/import/events      # 批量导入事件数据
POST   /api/v1/import/relations   # 批量导入关系数据
GET    /api/v1/import/template/{entity_type} # 获取导入模板
```

### 高级功能

#### 过滤和搜索
所有列表接口都支持以下过滤参数：
- **分页**: `skip`（跳过记录数）, `limit`（返回记录数）
- **搜索**: `search`（关键词搜索）
- **排序**: `order_by`（排序字段）, `order_desc`（是否降序）

#### 数据导入
- **支持格式**: CSV、JSON
- **批量处理**: 支持大批量数据导入
- **数据验证**: 自动验证数据格式和完整性
- **错误处理**: 详细的错误报告和部分成功导入

## 快速开始

### 1. 环境准备
```bash
# 确保Python 3.8+已安装
python --version

# 安装依赖
pip install -r requirements.txt
```

### 2. 初始化数据库
```bash
python init_db.py
```

### 3. 启动API服务
```bash
python main.py
```

### 4. 访问API文档
打开浏览器访问：http://localhost:8000/docs

## 数据导入指南

### 1. 获取导入模板
```bash
curl http://localhost:8000/api/v1/import/template/country
```

### 2. 准备数据文件
参考 `data/` 目录下的示例文件：
- `sample_countries.csv`: 国家数据示例
- `sample_wars.csv`: 战争数据示例

### 3. 导入数据
```bash
# 导入国家数据
curl -X POST -F "file=@data/sample_countries.csv" http://localhost:8000/api/v1/import/countries

# 导入战争数据
curl -X POST -F "file=@data/sample_wars.csv" http://localhost:8000/api/v1/import/wars
```

### 4. 验证导入结果
```bash
# 检查国家列表
curl http://localhost:8000/api/v1/countries/

# 检查战争统计
curl http://localhost:8000/api/v1/wars/stats
```

## 扩展开发

### 添加新的数据实体
1. 在 `models/` 目录创建新的模型文件
2. 在 `schemas/` 目录创建对应的Pydantic模式
3. 在 `crud/` 目录创建CRUD操作
4. 在 `routers/` 目录创建API路由
5. 在 `main.py` 中注册新路由

### 添加新的分析功能
1. 在 `routers/` 目录创建新的分析路由
2. 实现统计分析逻辑
3. 添加相应的数据模式

### 性能优化
1. 添加数据库索引
2. 实现缓存机制
3. 优化查询语句
4. 添加分页和过滤

## 部署说明

### 开发环境
- 数据库: SQLite3（文件存储）

- 端口: 8000

### 生产环境
建议迁移到PostgreSQL，使用Gunicorn + Nginx部署。

## 故障排除

### 常见问题

1. **数据库连接失败**
   - 检查数据库文件权限
   - 确保SQLite3正常安装

2. **API服务启动失败**
   - 检查端口是否被占用
   - 确保所有依赖已安装

3. **数据导入失败**
   - 检查文件格式
   - 验证数据完整性
   - 查看错误详情

4. **查询性能问题**
   - 添加适当的索引
   - 优化查询条件
   - 使用分页

API服务运行时会输出详细的日志信息，包括：
- 请求处理日志
- 数据库操作日志
- 错误信息
